hexo + github pages搭建个人博客

一直都想搭建一个自己的博客使用，一是为了记录自己的学习生活内容；二是为了通过博客来练习自己的技能。之前因为对这方面了解甚少，并也没有深入研究的时间，就一直搁置着。这俩天国庆假期间有了大把的时间，于是也是通过网络帮助学习，搭建了这一个简单的个人博客。

环境搭建

所用技术工具简单介绍：

  • git，免费、开源的分布式项目版本控制系统，在本项目里，通过与github的配合使用，来实现了项目代码的管理以及网络部署发布。
  • node.js，JavaScript运行环境，一个能够跑JavaScript项目的后台服务器，本项目是在node.js下运行。
  • Github，软件项目的托管平台，只支持 Git 作为唯一的版本库格式进行托管。github pages可以被认为是用户编写的、托管在github上的静态网页，它有轻量级、空间免费稳定、无需搭建服务器和可以绑定独立域名等优点，适合我们建立个人博客系统。
  • hexo，一个简单地、轻量地、基于Node的一个静态博客框架，可以方便的生成静态网页托管在github上。

本人所使用系统为WIN10，以下所有操作均是在WIN10环境下进行。

git、node.js安装

git安装，直接官网下载安装。

nodejs安装，通过官网下载安装，安装后可通过命令行node --version查看node.js版本并确认安装成功。

配置Github

如果没有github账号，通过官网注册。

配置SSH keys

配置SSH keys的主要目的是让本地git项目和远程github建立联系。

首先需要检查你电脑上现有的ssh keys，在git bash终端输入：

$ cd ~/.ssh #检查本机的ssh密钥

如果提示：No such file or directory 说明你是第一次使用git。否则，说明之前使用过git,ssh密钥在本地电脑上存在。

生成新的SSH key

如果本地没有ssh密钥，则需要生成新的密钥，终端输入：

$ ssh-keygen -t rsa -C "邮件地址@youremail.com"
Generating public/private rsa key pair.
Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<回车就好>

然后会提示输入加密串：

Enter passphrase (empty for no passphrase):<输入加密串>
Enter same passphrase again:<再次输入加密串>

加密串会在提交项目时使用，如果为空的话提交项目时则不用输入。这个设置是防止别人往你的项目里提交内容。

成功后页面如下：

添加ssh key到github

在本机设置SSH Key之后，需要添加到GitHub上，以完成SSH链接的设置。

本地查找用id_rsa.pub文件（一般在路径C:\Users\yourUserName.ssh），文本编辑工具打开，如果看不到这个文件，你需要设置显示隐藏文件。准确的复制这个文件的内容，才能保证设置的成功。

打开github网站，找到setting->SSH and GPG keys->New SSH key:

将id_rsa.pub黏贴到key框中并保存。

ssh key设置测试
在终端输入指令，看看设置是否成功(git@github.com的部分不要修改)：

$ ssh -T git@github.com

期间会输入加密串密码，输入后如显示：Hi XXX! You've successfully authenticated, but GitHub does not provide shell access.说明设置成功。

Hexo创建个人博客

简单博客创建

hexo安装简单，直接在命令行输入指令：

$ npm install -g hexo-cli #使用 npm 安装 Hexo。

就能完成安装。安装后通过以下命令来初始化hexo:

$ hexo init <folder>
$ cd <folder>
$ npm install

启动一个本地服务：

$ hexo server #或者hexo s

默认情况下，地址为http://localhost:4000/。限制就可以通过该地址访问到hexo blog页面。

2.2 hexo主题更改

hexo有许多主题可供选择，我们可以在官网的主题页面选择喜欢的主题，主题均是在github的开源项目。我所选用的主题为landscape-plus，我们即可以通过每个项目下的提供的指令安装

# 在hexo根目录下执行
git clone https://github.com/xiangming/landscape-plus.git themes/landscape-plus

也可以直接下载后，放在你本地hexo项目下的themes文件夹。

修改_config.yml配置，将theme属性修改为新的theme名称：

# Extensions
## Plugins: https://hexo.io/plugins/
## Themes: https://hexo.io/themes/
theme: landscape-plus

hexo部署到github

github pages配置

github创建新的仓库，仓库名称必须为username.github.io(username是你的github用户名，否则github page将不会正常工作)。

hexo操作

修改_config.yml文件：

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repository: https://github.com/username/username.github.io.git
  brance: master

执行指令：

$ hexo g #生成
$ hexo d #部署

如果出现Deployer not found:git，说明deployer插件没装，需要安装插件，执行指令：

npm install hexo-deployer-git --save

执行完成后，github仓库中会出现你所部署的代码。通过在浏览器中输入username.github.io.（注意io后面有“.”）。就可以浏览到你的页面。

设置独立域名

通过以上的操作，我们可以通过“username.github.io.”来访问到博客页面，但该地址过长且不易记忆。我们可以通过自己申请域名，来绑定该页面。

域名注册

域名注册需要通过域名注册商来获取，我是通过godaddy购买，godaddy域名优惠码挺多，一个域名第一年是20元，总体算来是比较经济实惠的；它还有一个12美元开一年虚拟主机并免费赠送一个域名的活动(这一方面有空写一文章另讲)，算下来买一年虚拟主机外加一年域名只需要七十多人民币。

DNS域名解析

由于godaddy是国外网站，域名解析服务器经常被强，不稳定，所以我选择使用国内的服务商DNSPod提供域名解析服务。
DNSpod域名解析服务是免费的，注册登录后，通过以下步骤来配置：

  • 修改DNS域名服务，登录godaddy，在“我的产品”中可以查看到域名，点击“管理”进入域名管理页面

进入后修改域名服务器类型，选择“自定义”添加两个域名服务器：

f1g1ns1.dnspod.net

f1g1ns2.dnspod.net

  • DNSPod添加域名解析。点击DNSPod左侧“域名解析”，右侧点击“添加域名”，将新申请的域名填写上。

  • DNSPod添加域名记录，在DNSPod自己的域名下添加一条A记录，地址就是你的Github Pages页面的IP，可以通过命令行ping查看

    ping username.github.io.

  • 修改后，域名解析并不会立即生效，大概会有24小时左右的缓冲时间，我们也可以通过ping来查看域名是否生效ping newname.com。查看IP是否为我们修改后的ip。

    域名与github pages绑定

    登录github，找到我们的博客项目仓库，点击”Settings”，修改coutom domain

修改完成后，如果域名解析已经生效，则可以直接通过我们自定义的域名访问到博客：

自动扫描组件<context:component-scan>使用说明

 2016年11月16日

在xml配置了这个标签后，spring可以自动去扫描base-pack下面或者子包下面的Java文件，如果扫描到有@Component @Controller@Service等这些注解的类，则把这些类注册为bean。

注意：如果配置了那么标签就可以不用再xml中配置了，因为前者包含了后者。另外还提供了两个子标签

在说明这两个子标签前，先说一下有一个use-default-filters属性，该属性默认为true,这就意味着会扫描指定包下的全部的标有@Component的类，并注册成bean。也就是@Component的子注解@Service,@Reposity等。所以如果仅仅是在配置文件中这么写

<context:component-scan base-package="tv.huan.weisp.web"/>

Use-default-filter此时为true那么会对base-package包或者子包下的所有的进行java类进行扫描,并把匹配的java类注册成bean。
